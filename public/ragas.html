评分体系

主要关注：Ragas 各指标的定义、计算逻辑、分布统计与逐样本评分。综合得分 = 所选指标的算术平均值；低质量样本定义为均分 < 0.5。

指标定义与计算逻辑

上下文精确度

检索到的上下文中有多大比例是与问题直接相关的。衡量检索系统的精确度——返回结果中有效信息占比越高，说明检索噪声越低。

计算公式
Precision@k = (相关上下文数量) / (检索到的上下文总数)
取值范围
0-1，越高表示检索精确度越好
低分影响
低精确度意味着检索返回大量无关文档，增加 LLM 处理负担并可能导致答案偏差
上下文召回率

检索到的上下文是否覆盖了回答问题所需的所有关键信息。衡量检索系统的召回能力——是否遗漏了必要的知识片段。

计算公式
Recall = (检索到的相关信息数量) / (回答所需的总信息数量)
取值范围
0-1，越高表示信息覆盖越完整
低分影响
低召回率意味着关键信息未被检索到，LLM 只能基于不完整的上下文生成答案
忠实度

生成的答案是否忠实于检索到的上下文，不引入外部知识或产生幻觉。该指标通过逐条验证答案中的声明是否可从上下文推导来评估。

计算公式
Faithfulness = (可由上下文支撑的声明数) / (答案中的总声明数)
取值范围
0-1，越高表示幻觉风险越低
低分影响
低忠实度说明 LLM 在生成中引入了上下文未涵盖的信息，存在事实错误风险
答案相关性

生成的答案与用户问题的切题程度。通过反向生成问题并计算与原始问题的语义相似度来衡量答案是否直接回应了用户意图。

计算公式
通过 Embedding 计算反向生成问题与原始问题的平均余弦相似度
取值范围
0-1，越高表示答案越切题
低分影响
低相关性说明答案偏离用户意图，可能包含冗余或不相关内容
